# AI Catalog Agent 回归检查清单

每次准备发包前，按本清单验证。目标是确认安装包可以真实交付，而不是只确认开发预览页可用。

## 自动检查

1. `npm.cmd test`
   - 验证翻译核心、授权核心、构建产物路径等单元测试。
2. `$env:VITE_LICENSE_ENDPOINT='http://43.129.236.16:8787/api/license/activate'; npm.cmd run build`
   - 验证前端和 Electron 生产资源可构建。
3. `npm.cmd run smoke:product`
   - 验证当前版本安装包存在、生产 HTML 使用相对资源、打包产物包含公网授权地址、包含字体上传桥接能力。
4. `$env:VITE_LICENSE_ENDPOINT='http://43.129.236.16:8787/api/license/activate'; $env:CSC_IDENTITY_AUTO_DISCOVERY='false'; npm.cmd run package:win`
   - 重新生成 Windows 安装包。

## 授权检查

1. 公网授权服务健康检查：`http://43.129.236.16:8787/health` 返回正常。
2. 生成月卡、年卡、永久三类激活码。
3. 月卡有效期从生成时间起 30 天，年卡从生成时间起 365 天，永久码不过期。
4. 使用激活码激活后，应用重启不需要再次输入激活码。
5. 授权到期后，应用提示输入新的激活码。
6. 同一激活码在新设备激活后，最新激活设备可用，旧设备校验失败。

## 桌面安装检查

1. 在本机安装最新 `release/AI-Catalog-Agent-Setup-<version>.exe`。
2. 启动后页面内容完整显示，不出现空白窗口。
3. 激活弹窗只要求填写激活码，不要求登录账号。
4. 模型配置切换服务商时，Endpoint 和模型名称同步切换。
5. 选择 `.ai` 文件按钮可打开本机文件选择器。
6. 输出位置支持源文件同目录、桌面、文档目录、自定义路径。
7. 结果区域支持打开输出文件和打开目录。

## Illustrator 翻译检查

1. 使用真实 `.ai` 文件跑一次完整任务。
2. 输出文件仍为可编辑 `.ai`。
3. 原图层、素材、整体排版不被破坏。
4. 英文文本被替换为目标语言。
5. 术语保护内容能影响翻译表达。
6. 任务中断或模型异常后，重新点击继续翻译会复用当前任务进度。
7. 关闭当前任务后，再选择文件会创建新任务，不复用旧任务进度。

## 字体检查

1. 不上传字体时，使用默认字体回写。
2. 上传 `.ttf`、`.otf`、`.ttc` 或 `.otc` 字体后，任务使用该字体优先回写。
3. 字体不可用或 Illustrator 找不到字体时，任务应回退默认字体，不阻断整体翻译。

## 新电脑检查

1. 在未安装开发环境的新 Windows 电脑上安装最新包。
2. 启动后页面内容完整显示。
3. 配置用户自己的模型 API Key。
4. 输入有效激活码完成激活。
5. 选择 `.ai` 文件并完成一次真实输出。
